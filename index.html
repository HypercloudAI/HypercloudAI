About the project

Hypercloud Intelligence is a research and engineering initiative created and led by developer Ariyan Ahmed (online: BloxyWave). The project focuses on compact model architecture, aggressive optimization, and production-ready packaging for Android and edge devices.

The engineering goals include efficient architectures, custom quantization and pruning pipelines, runtime optimizations, and reproducible tooling for evaluation. Emphasis is on enabling powerful on-device inference while keeping compute and monetary costs minimal.

Declaration
We are working to make an AI model without the help of OpenAI, Gemini, or any external LLM. See Wikipedia: Large language model for background on the term. Our objective is to create powerful AI that is free to use, deployable locally, and accessible worldwide.

Featured projects

Krix alpha
The first mobile-primed release: extremely optimized for Android and local inference. Minimal memory footprint and fast cold-start times; intended for low-resource platforms. Strengths: battery-friendly, tiny disk use; limitations: intentionally weak model capacity.
Dragonpy lite
A compact transformer variant prioritizing runtime efficiency and compatibility. Target: on-device assistants, game AI, and local inference on constrained hardware after quantization.
Dragonpy 1.0
The first full model release in the Dragonpy family. Designed for local inference with a balanced tradeoff between capability and resource cost. Reliable for offline tasks and prototype deployments.
Dragonpy Max
A cutdown, Android-only derivative of the Sonic line — optimized for real-world mobile throughput with aggressive pruning while preserving practical capability for edge services.
Dragonpy sonic
The primary high-performance model being developed from scratch. Focuses on scalable architecture, expandable tokenization, and efficient training pipelines. Still under active development with a strong emphasis on open research.
Dragonpy sonic (gpt4all)
A historical variant that used external assistance and community models during prototyping. Status: discontinued — retained for archival and learning purposes.

Projects & technical notes

Krix alpha
- Focus: extreme mobile optimization and minimal compute overhead.
- Approach: architecture simplification, aggressive quantization, on-device runtime micro-optimizations.
- Use cases: embedded assistants, telemetry filtering, lightweight decision agents.
Dragonpy 1.0
- Focus: robust, local inference with predictable resource usage.
- Approach: compact attention blocks, tokenization tuned for low-vocabulary domains, and deterministic runtime behavior.
- Use cases: offline productivity tools, education assistants, robotics control prototyping.
Dragonpy lite
- Focus: smallest practical footprint at acceptable capability.
- Approach: mixed precision quantization, layer fusion, and optimized inference kernels for ARM devices.
- Use cases: games, local chat companions, in-app microservices.
Dragonpy Max
- Focus: Android-only, throughput-oriented variant.
- Approach: selective head pruning, runtime batching, and optimized token pipelines for JNI bindings.
- Use cases: on-device summarization, edge NLP services in mobile apps.
Dragonpy sonic
- Focus: flagship model being engineered from first principles.
- Approach: modular attention shards, efficient tokenizer, and training recipes that prioritize parameter efficiency.
- Current status: active research; benchmarking and safety audits ongoing.
Dragonpy sonic (gpt4all) — discontinued
- Note: an archived experimental branch that used external community models during iteration. It helped seed design decisions but will not be continued as part of the main project.
Tooling & pipeline
We build reproducible pipelines for quantization, pruning, benchmarking, and Android packaging. Emphasis on open formats, explainable results, and efficient inference stacks.
Community & governance
The project intends to be community driven. Contributions will be evaluated for safety, reproducibility, and license compatibility. Research notes will be published to foster collaboration.

Technical goals

  • Maximize compute efficiency per parameter.
  • Enable on-device inference with low power draw.
  • Open toolchains for quantization and deployment.
  • Rigorous benchmarking and reproducible tests.

Ethics & safety

Research and release will follow best practices for responsible AI. Safety testing, adversarial analysis, and access controls are part of the roadmap. Prioritizing user privacy and transparency.
